lowMemory

If true, any ImageViews that are not in the immediate viewing area (i.e., the currently active image and the first image on either side of it) will be destroyed to free up memory. This has the benefit of using minimal memory (which is good for mobile devices), but has the downside that, if you want to view the images again, the ImageViews will have to be re-created and the images re-fetched (thus increasing the number of image-related GET calls). Defaults to false.
